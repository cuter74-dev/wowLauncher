# Work Log — 2026-06-22 (Part 2: features, i18n, branding)

## Plan
- Desktop: register an installed application in one step (auto name & icon).
- Desktop & mobile: 11-language localization + a language setting.
- Theme/icon/splash matching the logo (cyan/dark); unify the app name as `wowLauncher`.

## Work done
- **Add from Applications**: the file picker can't select macOS `.app` bundles, so we scan the install folders (`/Applications`, etc.) and pick from an in-app list. On selection, `Info.plist` + `sips` extract the `.icns` → PNG to auto-fill name & icon. Icon thumbnails are shown in the list and picker (extracted lazily for visible rows + cached).
- **11-language localization**: `flutter_localizations` + `intl` + ARB (desktop 70 keys, mobile 23 keys × 11). English/Korean/Spanish/Chinese/Japanese/Russian/French/German/Vietnamese/Indonesian/Hindi. Desktop Settings dropdown and a new mobile Settings screen; instant switching + DB persistence (mobile settings table v2 migration).
- **Branding**: cyan (`0xFF22D3EE`) + dark theme, in-app logo (desktop rail / mobile AppBar). App name `wowLauncher` (mobile label / desktop `PRODUCT_NAME`). Mobile launcher icon & splash (`flutter_native_splash`), desktop macOS icon.
- On-device check: wireless adb failed (cable/network) → switched to USB and verified install/run on a Galaxy A51. Desktop verified running/launching on macOS.
- Committed & pushed to GitLab `main` (Korean) and GitHub `main` (English).

## Approach
- Generated localization code (`app_localizations*.dart`) is gitignored (regenerated on build).
- Native artifacts (app name/icon/splash/permissions) live in git-ignored runner folders → only the `pubspec` `flutter_launcher_icons`/`flutter_native_splash` config and ARB are committed; other machines regenerate via `flutter create` then `dart run`.
- Code changes are committed on `main`, then cherry-picked to `github-publish` and amended with an English message.

## Result
- Pushed GitLab `main` `2a966d4`, GitHub `main` `64ea31f`.
- Verified language switching, logo, and app name on both desktop (macOS) and mobile (Android).
